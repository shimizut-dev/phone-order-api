# 用語集

本書は `phone-order-api` のドメインで扱う主要な用語を整理する。  
現行コードに存在する概念を対象とし、実装と用語の対応を明確にする。

---

## 概要

- 現行ドメインは注文を中心に構成されている
- 注文は複数の値オブジェクトと状態で表現される
- 更新時の整合性はバージョンで管理する

---

## 内容

### 方針

- 用語は現行コードに存在するクラス名と役割を基準に整理する
- 業務用語と実装用語がずれないように対応関係を明記する
- 未実装の概念は断定せず、現状をそのまま記載する

### ルール

| 用語      | 実装上の対応               | 説明                  |
|---------|----------------------|---------------------|
| 注文      | `Order`              | 注文全体を表す集約ルート        |
| 注文ID    | `OrderId`            | システム内部で利用する一意識別子    |
| 注文コード   | `OrderCode`          | 外部向け識別子。`ORD` + 6 桁 |
| 注文日時    | `OrderedAt`          | 注文が登録された日時          |
| 注文ステータス | `OrderStatus`        | 注文の状態を表すコードと名称      |
| バージョン   | `Version`            | 楽観的ロック用の更新管理値       |
| 注文リポジトリ | `OrderRepository`    | 注文の取得・登録・更新を担う境界    |
| 注文コード採番 | `OrderCodeGenerator` | 新規注文コードを生成する境界      |

### 方針上の補足

- 注文の外部向け識別は `OrderCode` を正本とする
- `OrderId` は内部識別子であり、外部 API では返却しない
- 注文ステータスは名称ではなくコードで管理する
- バージョンは更新競合の検知に使用する

### 手順

1. 新しいドメイン概念を追加する場合は、既存用語と重複しないか確認する
2. 実装に新しい値オブジェクトや境界を追加した場合は本書に反映する
3. 外部向けと内部向けで用語の使い分けがある場合は両方を明記する
