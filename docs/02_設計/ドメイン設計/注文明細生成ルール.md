# 注文明細生成ルール

本書は注文明細生成に関する現状を整理する。  
現行コードには注文明細を表すドメインモデルが存在しないため、未実装事項として扱う。

---

## 概要

- 現行実装には注文明細を保持するモデルが存在しない
- 注文集約は注文ヘッダ相当の情報のみを管理している
- 注文明細生成ルールは今後の拡張対象である

---

## 内容

### 方針

- 現時点では注文明細に関する業務ルールを実装済みとはみなさない
- 注文明細を追加する場合は `Order` 集約との境界を先に定義する
- 既存の注文登録 API は注文明細入力を受け取らない前提で扱う

### ルール

- 現行の注文登録で必須なのは注文日時のみである
- 注文明細数、明細金額、商品情報、契約情報は管理対象外である
- 注文明細生成の責務を持つドメインサービスや値オブジェクトは現時点で存在しない

### 手順

1. 注文明細を扱う要件が発生した場合は、まず `01_要件` にスコープと業務ルールを追加する
2. 次に集約境界、明細モデル、生成責務を `02_設計/ドメイン設計` に追加する
3. API 契約と永続化設計を追従させる
