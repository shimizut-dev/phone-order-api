# 命名ルール

本書は `phone-order-api` のデータベース命名ルールを整理する。  
現行の migration と JPA entity 実装を基準とする。

---

## 概要

- テーブル、カラム、制約、インデックスの命名を統一する
- 実装済みの DB オブジェクト名を正本として扱う
- 将来のテーブル追加時も同じルールで拡張できるようにする

---

## 内容

### 方針

- 物理名は英小文字のスネークケースを基本とする
- 実装済みの `orders` テーブルと制約名を正とする
- 略語は必要最小限に留め、意味が分かる名前を使う

### ルール

- テーブル名は英小文字の複数形名詞を基本とする
- カラム名は英小文字のスネークケースとする
- 主キーカラム名は `id` に統一する
- 主キー制約名は `pk_<table_name>` とする
- 一意制約名は `uk_<table_name>_<column_name>` とする
- インデックス名は `idx_<table_name>_<column_name>` を基本とする
- 監査カラムは `created_at`, `created_by`, `updated_at`, `updated_by`, `deleted_at`, `deleted_by` に統一する
- 業務向けコードのカラム名は `order_code` のように `<対象>_code` を基本とする
- `order_code` の値形式は `ORD000001` のようにハイフンなし 9 文字とする
- ステータスコードのカラム名は `order_status` のように意味の分かる名前を使う

現行の主な実例:

```text
orders
id
order_code
ordered_at
order_status
version
pk_orders
uk_orders_order_code
order_code_seq
next_order_code
```

### 手順

1. 新しいテーブルやカラムを追加する場合は、既存の `orders` 命名と揃える
2. 制約やインデックスを追加する場合は、種別接頭辞と物理名を組み合わせて命名する
3. JPA entity の `@Table`、`@Column`、migration の物理名を一致させる
