# データベース概要

本書は `phone-order-api` の現行データベース構成を整理する。  
現行の Flyway migration と JPA 実装を基準とする。

---

## 概要

- DB 製品は PostgreSQL 16 系を前提とする
- スキーマ変更は Flyway migration で管理する
- 現行の主要テーブルは `orders` である

---

## 内容

### 方針

- DDL の正本は `src/main/resources/db/migration` 配下の Flyway migration とする
- JPA entity と migration の整合を保つ
- 共通監査カラムと論理削除カラムは DB 設計で明示する

### ルール

- ローカル開発とテストで PostgreSQL 16 系を利用する
- migration は `V1` から `V4` まで適用され、`orders` テーブル、注文コード一意制約、採番シーケンス、バージョン列を定義する
- `orders` テーブルの主要カラムは `id`, `order_code`, `ordered_at`, `order_status`, `version` である
- 共通監査カラムは `created_at`, `created_by`, `updated_at`, `updated_by`, `deleted_at`, `deleted_by` とする
- 論理削除カラムは保持するが、現行ユースケースに削除機能は存在しない
- `order_code` には `uk_orders_order_code` の一意制約を付与する

`orders` テーブルの現行カラム順:

```text
id
order_code
ordered_at
order_status
version
created_at
created_by
updated_at
updated_by
deleted_at
deleted_by
```

### 手順

1. DB 変更が必要な場合は新しい Flyway migration を追加する
2. 必要に応じて JPA entity、mapper、repository 実装を更新する
3. 変更後は ER 図と関連資料も更新する
