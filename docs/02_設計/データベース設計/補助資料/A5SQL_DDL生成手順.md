# A5SQL_DDL生成手順

本書は A5:SQL Mk-2 を利用して DDL を確認または生成する際の手順を整理する。  
DDL の正本は Flyway migration とし、本書は補助資料として扱う。

---

## 概要

- A5:SQL Mk-2 利用時の設定方針を整理する
- 自動生成できる部分と手動確認すべき部分を分ける
- Flyway migration と差分が出ないようにする

---

## 内容

### 方針

- DDL の正本は `src/main/resources/db/migration` とする
- A5:SQL Mk-2 は ER 図確認や DDL 補助生成のために使う
- 生成後は制約名やコメントの妥当性を必ず確認する

### ルール

- RDBMS 種別は PostgreSQL を選択する
- 外部キー制約はリレーションシップから生成する
- 識別子をダブルクォートで囲まない
- 主キー制約名は `pk_%0:s` を基本とする
- インデックス名は `idx_%0:s_%1:d` を基本とする
- 自動生成された外部キー名は、必要に応じて実カラム名ベースへ手修正する
- 生成結果をそのまま正本にせず、Flyway migration と一致しているか確認する

### 手順

1. A5:SQL Mk-2 で ER 図を開く
2. PostgreSQL 向け設定で DDL を生成する
3. テーブル名、カラム名、制約名、コメントを確認する
4. 正式な変更は Flyway migration として追加する
