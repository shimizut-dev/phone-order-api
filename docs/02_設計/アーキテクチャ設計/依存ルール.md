# 依存ルール

本書は `phone-order-api` のレイヤ間依存ルールを整理する。  
現行実装と `ArchitectureTest` の検証内容を基準とする。

---

## 概要

- 依存方向を一方向に揃えて変更影響を局所化する
- domain と application は外側の技術詳細へ依存しない
- 一部ルールは ArchUnit テストで自動検証する

---

## 内容

### 方針

- 依存方向は外側から内側を基本とする
- domain は最内層とし、他レイヤやフレームワークへ依存しない
- application は domain に依存し、presentation と infrastructure には依存しない
- infrastructure は domain 境界の実装として内側へ接続する

### ルール

- `presentation` は `application` を利用してユースケースを実行する
- `application` は `domain` 型と `domain` 境界だけを利用する
- `domain` は `application`、`presentation`、`infrastructure` に依存しない
- `domain` は `org.springframework..`、`jakarta.servlet..`、`jakarta.persistence..` に依存しない
- `application` は `presentation` と `infrastructure` に依存しない
- `presentation` から JPA エンティティや repository 実装を直接扱わない
- `infrastructure` から controller や OpenAPI 生成モデルへ依存しない

### 手順

1. 新しいクラスを追加する場合は、依存先がレイヤルールに収まるか確認する
2. repository や採番のような境界が必要な場合は、契約を domain に置く
3. 具体実装は infrastructure へ配置する
4. ルールを追加または変更する場合は `ArchitectureTest` の検証対象も見直す
