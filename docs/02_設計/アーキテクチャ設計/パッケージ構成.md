# パッケージ構成

本書は `phone-order-api` の現行 Java パッケージ構成を整理する。  
実際に存在するパッケージと生成コードの配置を基準とする。

---

## 概要

- ルートパッケージと主要サブパッケージを整理する
- 各パッケージに何を置くかを明確にする
- 現行実装に存在しない概念は記載しない

---

## 内容

### 方針

- パッケージはレイヤ単位を基本とする
- ドメイン上の中心は `order` パッケージとする
- 生成物と手書きコードは役割を分けて配置する

### ルール

ルートパッケージ:

```text
jp.co.shimizutdev.phoneorderapi
```

現行構成:

```text
jp.co.shimizutdev.phoneorderapi
├─ application
│  └─ order
├─ domain
│  ├─ common
│  └─ order
├─ infrastructure
│  ├─ codegen
│  ├─ config
│  ├─ log
│  ├─ persistence
│  │  └─ order
│  └─ repository
│     └─ order
└─ presentation
   ├─ error
   ├─ log
   ├─ order
   └─ generated
      ├─ api
      └─ model
```

- `application.order`
  - `OrderService` を配置する
- `domain.common`
  - `PageResult`、`PagingCondition` を配置する
- `domain.order`
  - `Order` 集約、値オブジェクト、例外、repository interface、採番 interface を配置する
- `presentation.order`
  - controller と request/response mapper を配置する
- `presentation.error`
  - API エラーレスポンスと例外ハンドラを配置する
- `presentation.log`
  - `TraceIdFilter`、`RequestResponseLogFilter` を配置する
- `presentation.generated`
  - OpenAPI から生成された API interface と model を配置する
- `infrastructure.persistence.order`
  - JPA entity、JPA mapper、Spring Data repository を配置する
- `infrastructure.repository.order`
  - domain repository 実装を配置する
- `infrastructure.codegen`
  - `OrderCodeGenerator` の実装を配置する

### 手順

1. 新しいクラスを追加する場合は、まず該当レイヤを決める
2. 次に `order` のような機能単位サブパッケージへ収まるか確認する
3. 生成物に相当するものは手書きパッケージへ混在させない
