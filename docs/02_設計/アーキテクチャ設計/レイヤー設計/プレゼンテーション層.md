# プレゼンテーション層

本書は `phone-order-api` のプレゼンテーション層を整理する。  
現行の controller、mapper、filter、例外ハンドラ実装を基準とする。

---

## 概要

- HTTP リクエストを受け付け、レスポンスを返す
- OpenAPI 生成モデルと内部ドメインモデルの橋渡しを行う
- 例外を共通エラーレスポンスへ変換する

---

## 内容

### 方針

- HTTP 固有処理はプレゼンテーション層に閉じ込める
- ビジネスロジックは application と domain に委譲する
- OpenAPI 生成 API interface と model を境界モデルとして扱う

### ルール

- `OrderController` は `OrdersApi` を実装する
- 入力値の文字列や JSON は `OrderRequestMapper` で domain 型へ変換する
- domain の結果は `OrderResponseMapper` で API レスポンスへ変換する
- `ApiExceptionHandler` で domain/application/infrastructure 由来の例外を HTTP エラーへ変換する
- `TraceIdFilter` は `X-Request-Id` または `X-Correlation-Id` を優先して traceId を決定し、レスポンスヘッダー `X-Trace-Id` へ返す
- `RequestResponseLogFilter` はリクエスト、レスポンス、本文、ヘッダーをログ出力し、`LogMasker` でマスキングする
- JPA エンティティや repository 実装を直接扱わない

### 手順

1. 新しい API を追加する場合は、OpenAPI 生成 interface に対応する controller 実装を追加する
2. API 入力は mapper で domain 型へ変換する
3. API 出力は mapper で生成 model へ変換する
4. 新しい例外が利用者向けに返る場合は `ApiExceptionHandler` と OpenAPI 仕様を更新する
