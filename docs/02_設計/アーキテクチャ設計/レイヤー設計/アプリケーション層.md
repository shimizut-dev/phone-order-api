# アプリケーション層

本書は `phone-order-api` のアプリケーション層を整理する。  
現行の `OrderService` 実装を基準とする。

---

## 概要

- ユースケースの処理順序を制御する
- トランザクション境界を持つ
- domain 境界を利用して注文の取得、作成、更新を行う

---

## 内容

### 方針

- 業務上の正しさは domain に委ね、アプリケーション層は処理の流れを制御する
- 引数と戻り値は可能な限り domain 型で扱う
- HTTP、SQL、JPA には依存しない

### ルール

- 現行のアプリケーションサービスは `OrderService` である
- `getOrders` は `PagingCondition` を受けて `PageResult<Order>` を返す
- `getOrderByOrderCode` は `OrderCode` を受けて注文を取得する
- `createOrder` は `OrderCodeGenerator` で採番し、`Order.create` で注文を生成して保存する
- `cancelOrder` は注文取得後に `Order.cancel` を呼び出し、更新結果を保存する
- 参照系は `@Transactional(readOnly = true)`、更新系はメソッド単位で `@Transactional` を付与する
- OpenAPI 生成モデルや JPA エンティティを直接扱わない

### 手順

1. 新しいユースケースを追加する場合は、必要な domain 境界を洗い出す
2. 入力は presentation で domain 型へ変換してから受ける
3. 集約の生成や状態変更は domain の振る舞いへ委譲する
4. 永続化は repository interface 経由で行う
