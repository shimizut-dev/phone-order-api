# アーキテクチャ設計

本ディレクトリは `phone-order-api` のアーキテクチャ資料を管理します。  
レイヤ構成、依存ルール、パッケージ構成を整理します。

---

## 概要

- アーキテクチャ全体像と採用理由を共有する
- レイヤ責務と依存方向を明確にする
- 実装パッケージとの対応関係を整理する

---

## 管理資料

- `レイヤー構成概要.md`
  - 各レイヤーの主要クラス、依存関係、データフローを図で整理する
- `アーキテクチャ概要.md`
  - 現行実装の全体構成とレイヤ境界を整理する
- `採用方針.md`
  - DDD と Clean Architecture の採用方針を整理する
- `依存ルール.md`
  - レイヤ間の依存方向と禁止事項を整理する
- `レイヤーとパッケージ対応.md`
  - 設計上のレイヤー用語と実装パッケージの対応を整理する
- `パッケージ構成.md`
  - 現行の Java パッケージ構成を整理する
- `レイヤー設計/`
  - プレゼンテーション層、アプリケーション層、ドメイン層、インフラ層の責務資料を配置する
