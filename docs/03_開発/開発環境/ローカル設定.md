# ローカル設定

本書は `phone-order-api` をローカルで動かすための設定を整理します。  
`.env.example`、`docker/docker-compose.yml`、`application-local.properties` を前提にします。

---

## 概要

- ローカル DB 接続と環境変数の設定方法を定義する
- IntelliJ や Maven から起動する際の前提を揃える
- 文字コード設定を含むローカル開発時の注意点を残す

---

## 内容

### 前提

- Docker で PostgreSQL を起動する
- ローカル実行時は `application-local.properties` を利用する
- 必要な環境変数は `.env.example` を基準に設定する

### ルール

- テキストファイルは UTF-8 を使用する
- UTF-8 BOM は使用しない
- エディタや IDE の保存設定は [.editorconfig](../../../.editorconfig) に合わせる

### 方針

- `.env.example` の初期値は `local / local`
- ローカル DB の接続先は `jdbc:postgresql://localhost:5432/phone_order_api_local_db`
- Docker Compose の PostgreSQL コンテナ名は `phone-order-api-local-db`

### 手順

1. `.env.example` を基に必要な環境変数を設定する
2. `docker/docker-compose.yml` で PostgreSQL を起動する
3. IntelliJ または Maven から `local` プロファイルで起動する
4. generated sources が未解決の場合は `./mvnw generate-sources` 実行後に Maven を再読込する

ローカル DB 起動:

```bash
docker compose -f docker/docker-compose.yml up -d
```
