# ローカル設定

本書は `phone-order-api` をローカルで起動するための設定を整理する。  
現行の `.env.example`、`docker/docker-compose.yml`、`application-local.properties` を基準とする。

---

## 概要

- ローカル DB 接続と環境変数の設定方法を明確にする
- IntelliJ や Maven から起動する前提を揃える
- よくある復旧手順への導線を用意する

---

## 内容

### 方針

- 秘密情報はソースに埋め込まず、環境変数で渡す
- ローカル DB は Docker Compose で起動する
- ローカル実行時は `application-local.properties` を利用する

### ルール

- 必要な環境変数は `PHONE_ORDER_DB_USERNAME` と `PHONE_ORDER_DB_PASSWORD` である
- `.env.example` の初期値は `local / local` である
- ローカル DB の接続先は `jdbc:postgresql://localhost:5432/phone_order_api_local_db` とする
- Docker Compose の PostgreSQL コンテナ名は `phone-order-api-local-db` とする
- Flyway の `baseline-on-migrate` はローカル設定で有効にしている

### 手順

1. `.env.example` を参考に必要な環境変数を設定する
2. `docker/docker-compose.yml` で PostgreSQL を起動する
3. IntelliJ ではローカルプロファイル設定を読み込んで起動する
4. generated sources が未解決の場合は `./mvnw generate-sources` 実行後に Maven プロジェクトを再読込する

ローカル DB 起動例:

```bash
docker compose -f docker/docker-compose.yml up -d
```
