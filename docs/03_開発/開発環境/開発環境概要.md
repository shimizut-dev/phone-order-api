# 開発環境概要

本書は `phone-order-api` の開発環境の前提を整理する。  
現行の `pom.xml`、`docker/docker-compose.yml`、`application.properties`、CI 設定を基準とする。

---

## 概要

- Java、Maven、PostgreSQL、Docker、IntelliJ を前提とする
- ローカル実行と CI の主要バージョン差分を明確にする
- ローカル DB とテスト DB の前提を揃える

---

## 内容

### 方針

- Java と Maven の最低バージョンは `pom.xml` の Enforcer 設定を正本とする
- ローカル DB は `docker/docker-compose.yml` の PostgreSQL 16 を基準とする
- CI は GitHub Actions 上で Java 21 と Debian Trixie Slim を利用する

### ルール

- Java は 21 を使用する
- Maven は 3.9.0 以上を使用する
- PostgreSQL は 16 系を使用する
- IntelliJ の追加プラグインとして `SonarQube for IDE`、`GitHub Copilot - Your AI Pair Programmer`、`Mermaid` を利用する
- ローカル DB 接続先は `application-local.properties` の `phone_order_api_local_db` を使用する
- Flyway migration は `src/main/resources/db/migration` を正本とする
- OpenAPI 生成物は `target/generated-sources/openapi` へ出力される
- 品質確認は `./mvnw spotless:check test` を基本とする
- Spring Boot Actuator を導入し、`/actuator/health` を運用監視用エンドポイントとして公開する
- Actuator の Web 公開範囲は `health` のみに制限する（`management.endpoints.web.exposure.include=health`）
- `/actuator/health` は認証不要で疎通確認に利用し、その他の管理エンドポイントは公開しない

利用する IntelliJ プラグイン:

- `SonarQube for IDE`
  - ローカルで静的解析指摘を確認する
- `GitHub Copilot - Your AI Pair Programmer`
  - 実装補助とコード補完に利用する
- `Mermaid`
  - Mermaid 記法の図を IDE 上で確認する

### 手順

1. Java 21、Maven、Docker、IntelliJ を用意する
2. `.env.example` を参考にローカル用の環境変数を設定する
3. `docker/docker-compose.yml` で PostgreSQL を起動する
4. IntelliJ または Maven でアプリケーションを起動する
