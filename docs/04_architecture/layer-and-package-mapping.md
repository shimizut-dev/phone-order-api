# DDDレイヤーと phone-order-api パッケージの対応表

## 目的

DDDで使うレイヤー用語と、phone-order-api の実装パッケージの対応関係を明確にする。

本資料では、設計上の用語と実装上のパッケージ名が完全一致しない箇所についても、
意味の違いではなく表現上の違いとして整理する。

---

## 結論

phone-order-api では、DDD / Clean Architecture の考え方を踏まえつつ、
実装パッケージは以下の名称を採用する。

- 設計上の用語：ユーザインタフェース層
- 実装上のパッケージ：presentation

この差異は許容する。
ただし、資料上で対応関係を明記し、読み手が迷わないようにする。

---

## レイヤー対応表

| 設計上の用語      | 実装上のパッケージ      | 主な責務                                           |
|-------------|----------------|------------------------------------------------|
| ユーザインタフェース層 | presentation   | Controller、Request/Response DTO、API入出力、例外の入口制御 |
| アプリケーション層   | application    | ユースケース実行、トランザクション境界、アプリケーションサービス               |
| ドメイン層       | domain         | エンティティ、値オブジェクト、集約、ドメインサービス、リポジトリIF             |
| インフラストラクチャ層 | infrastructure | DBアクセス、外部連携、Repository実装、永続化設定                 |

---

## 各レイヤーの考え方

### 1. ユーザインタフェース層 = presentation

DDDやクリーンアーキテクチャの文脈では「ユーザインタフェース層」と表現する。

一方、phone-order-api では実装パッケージ名として `presentation` を使用する。

理由

- `application / domain / infrastructure / presentation` の並びが分かりやすい
- Javaパッケージ名として `userinterface` より短く自然
- Spring Boot の Controller 配置先としても扱いやすい

そのため、本プロジェクトでは

- 設計上の用語：ユーザインタフェース層
- 実装上のパッケージ：presentation

として運用する。

### 2. application

application パッケージは、ユースケースの実行を担う。

主な責務

- ユースケース単位の処理
- ドメインオブジェクトの利用
- トランザクション制御
- 外部入出力とドメインの橋渡し

### 3. domain

domain パッケージは、業務ルールの中心を担う。

主な責務

- エンティティ
- 値オブジェクト
- 集約
- ドメインサービス
- リポジトリインタフェース

### 4. infrastructure

infrastructure パッケージは、技術的な詳細を担う。

主な責務

- JPAエンティティ
- Repository実装
- DBアクセス
- 外部サービス呼び出し
- フレームワーク依存処理

---

## 用語運用ルール

### 1. 資料では設計上の用語を優先する

資料上では以下の用語を基本とする。

- ユーザインタフェース層
- アプリケーション層
- ドメイン層
- インフラストラクチャ層

### 2. 実装説明ではパッケージ名を併記する

必要に応じて以下のように記載する。

- ユーザインタフェース層（presentation）
- アプリケーション層（application）
- ドメイン層（domain）
- インフラストラクチャ層（infrastructure）

### 3. 別名を増やしすぎない

以下のような表現の乱立は避ける。

- API層
- Controller層
- Web層
- UI層

必要がある場合でも、ユーザインタフェース層 / presentation に統一して説明する。

---

## 補足

本資料は、レイヤー設計とパッケージ設計の対応関係を説明するための資料である。

そのため、実装詳細や命名ルール全般ではなく、
「設計上の用語」と「実装上の表現」の橋渡しを目的とする。
