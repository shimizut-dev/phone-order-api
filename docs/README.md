# docs

本ディレクトリは `phone-order-api` のドキュメント群を管理する。  
日本語ディレクトリを正本として、要件、設計、開発、テスト、ADR を用途ごとに分けて整理する。

---

## 概要

- ドキュメントの配置場所を明確にする
- 設計判断の参照先を統一する
- 実装とドキュメントのずれを減らす
- README から詳細資料へ辿りやすくする

---

## 管理資料

- `00_はじめに/README.md`
  - `docs/` 全体の入口
- `00_はじめに/ドキュメントガイドライン.md`
  - ドキュメント全体の方針、命名、配置、テンプレート利用ルール
- `01_要件/README.md`
  - 要件整理の入口
- `02_設計/README.md`
  - 設計資料の入口
- `03_開発/README.md`
  - 開発ガイドの入口
- `05_ADR/README.md`
  - ADR の入口
