# スコープ

本書は `phone-order-api` が扱う業務範囲を整理する。  
現時点では、移動機注文の登録、参照、キャンセルに関する API を対象とする。

---

## 概要

- 本システムは注文を API 経由で管理する
- 外部公開 API は `/api/v1/orders` を起点とする
- 注文は `orderCode` を業務向け識別子として扱う
- 現行実装で扱う主機能は一覧取得、単票取得、登録、キャンセルである

---

## 内容

### 対象

- 注文一覧の取得
- 注文コードによる注文取得
- 注文日時を指定した注文登録
- 注文のキャンセル
- 注文状態と注文バージョンの管理
- PostgreSQL への注文データ永続化

### 対象外

- 注文内容の明細管理
- 顧客情報管理
- 商品管理
- 決済処理
- 出荷実行や配送連携
- 認証・認可機能
- 管理画面や画面 UI
- 注文状態の進行処理全体

### 取り扱う注文情報

- 注文コード
- 注文日時
- 注文ステータス
- 楽観的ロック用バージョン
- 監査項目
