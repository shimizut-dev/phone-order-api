# 関係者

本書は `phone-order-api` の利用者と関係者を整理する。  
現行実装には認証機能がないため、ここでは役割ベースで整理する。

---

## 概要

- 本 API は移動機注文を管理する業務システムから利用されることを想定する
- 主な利用者は注文を登録・参照・キャンセルする業務担当者である
- 開発・運用では API 仕様、DB、ログを扱う技術担当者も関係者となる

---

## 内容

### 利用者

#### 受注担当者

- 注文を登録する
- 注文コードで注文を確認する
- 必要に応じて注文をキャンセルする

#### 業務管理者

- 注文の状態を確認する
- 業務ルールに沿ってキャンセル可否を判断する
- 障害時に注文状況を確認する

### 技術関係者

#### API 利用システム開発者

- OpenAPI を参照してクライアント実装を行う
- 入出力形式とエラーレスポンスを確認する

#### バックエンド開発者

- API、ドメイン、永続化の実装を変更する
- 要件、設計、開発ガイドを更新する

#### 運用担当者

- ログや監査項目を用いて状況を確認する
- 想定外エラーや競合発生時の調査を行う

### 想定する利用シーン

- 注文受付時に注文を登録する
- 注文確認時に注文コードで検索する
- 注文状況の確認のために一覧を参照する
- 状態とバージョンを確認した上で注文をキャンセルする
