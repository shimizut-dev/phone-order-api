# ADR-0005: Testcontainers を採用する

## ステータス
採用

## 日付
2026-04-04

---

## コンテキスト

phone-order-api プロジェクトでは、データベースを使用したテストを実施する必要がある。

テスト方法として以下を検討した。

- Testcontainers
- H2（インメモリDB）
- ローカルDB（PostgreSQL）

検討観点

- 実務に近いテスト環境
- テストの信頼性
- テストの実行速度
- 環境構築の容易さ
- 保守性

---

## 決定

Testcontainers を採用する

---

## 理由

### 1. 本番に近い環境でテストが可能

Testcontainers を使用することで、
Docker 上に PostgreSQL を起動してテストを実行できる。

これにより、本番環境に近い条件でテストが可能となる。

---

### 2. DB 方言の違いを防げる

H2 を使用した場合

- SQL 方言の違い
- インデックスの挙動の違い
- NULL の扱いの違い

などが発生する可能性がある。

Testcontainers では PostgreSQL を使用するため、
この問題を回避できる。

---

### 3. 実務での採用が増えている

Testcontainers は近年の Spring Boot プロジェクトで
広く採用されている。

実務想定のポートフォリオとして価値が高い。

---

### 4. CI/CD との相性が良い

Testcontainers は Docker ベースのため、
CI 環境でも同じ構成でテスト可能。

---

## H2 との比較

### H2 のメリット

- 軽量
- 起動が高速
- 設定が簡単
- Docker 不要

### H2 のデメリット

- PostgreSQL と挙動が異なる
- SQL 方言の違い
- 本番との差異が発生
- 本番で動かない SQL がテストで通る可能性

---

## ローカルDBとの比較

### ローカルDB のメリット

- 本番に近い環境
- Testcontainers より高速

### ローカルDB のデメリット

- 開発者環境に依存
- 環境構築が必要
- CI 環境での再現が難しい

---

## 影響

### ポジティブな影響

- 本番に近いテストが可能
- DB 差異による不具合を防げる
- CI との統一

### ネガティブな影響

- テスト実行時間が長くなる
- Docker が必要

---

## 代替案

### H2

メリット

- 軽量
- 高速

デメリット

- 本番との差異

---

### ローカルDB

メリット

- 本番に近い

デメリット

- 環境依存

---

## 結論

phone-order-api では Testcontainers を採用する。

本番に近い環境でテストを実行でき、
実務想定のプロジェクトとして適しているため。
