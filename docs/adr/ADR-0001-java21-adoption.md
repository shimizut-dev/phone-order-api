# ADR-0001: Java 21 を採用する

## ステータス
採用

## 日付
2026-04-04

---

## コンテキスト

phone-order-api プロジェクトでは Java を使用して Spring Boot ベースの Web API を開発する。

Java のバージョンとして以下の候補を検討した。

- Java 17 (LTS)
- Java 21 (LTS)

検討観点は以下の通り。

- 実務での採用状況
- 長期サポート（LTS）
- 学習価値
- 将来性
- Spring Boot の対応状況

---

## 決定

Java 21 を採用する。

---

## 理由

### 1. LTS（Long Term Support）であるため

Java 21 は LTS バージョンであり、長期サポートが提供される。  
実務でも安心して採用可能。

### 2. 最新の安定バージョンであるため

Java 21 は現時点での最新の LTS バージョンであり、  
今後のプロジェクトでも採用される可能性が高い。

### 3. ポートフォリオとしての価値が高い

最新技術を採用することで、以下のアピールが可能。

- 技術選定能力
- 最新技術への対応力
- 将来性を考慮した設計

### 4. Spring Boot が対応している

Spring Boot 3.x は Java 21 に対応しているため、  
技術的な問題はない。

### 5. 新機能を活用できる

Java 21 では以下の機能が利用可能。

- Virtual Threads
- Record Patterns
- Pattern Matching for switch
- Sequenced Collections

これらの機能は将来的に活用できる。

---

## 影響

### ポジティブな影響

- 最新技術の利用が可能
- 長期サポート
- ポートフォリオ価値向上

### ネガティブな影響

- 一部ライブラリの対応状況に注意が必要
- 実務で Java 17 が多い場合がある

---

## 代替案

### Java 17

メリット

- 実務採用が多い
- 安定している

デメリット

- 将来性がやや低い
- 最新機能が使えない

---

## 結論

phone-order-api では Java 21 を採用する。
