# ドキュメントガイドライン

`docs/` 配下で管理するドキュメントの基本方針を定義します。  
新規ドキュメントを作成する場合は、本書を基準に目的、配置先、ファイル名、テンプレートを決めます。

---

## 概要

- `00_はじめに` 配下の正本として `docs/` 全体の入口文書を管理する
- `docs/` では、要件、設計、開発、テスト、ADR を分けて管理する
- カテゴリ構成、命名、配置、テンプレート利用の上位方針をここで定義する
- Markdown 記法や文字コードなどの基本方針も本書で扱う

---

## 内容

### 方針

- 要件は「何を実現するか」を整理する
- 設計は「どう表現するか」を整理する
- ADR は「なぜその判断を採ったか」を残す
- 開発ガイドは「どう作業するか」を揃える

- `docs/` は読者と用途ごとにディレクトリを分けて管理する
- 上位ディレクトリには入口となる `README.md` を置く
- 詳細な説明は個別資料に分け、上位 README には導線だけを書く
- 同じ主題のドキュメントを複数箇所に重複配置しない
- 役割の異なる情報を 1 つのドキュメントに詰め込みすぎない
- 正本と補助資料の関係が分かるように構成する

### ルール

- 本書の対象は `docs/` 配下の README、要件、設計、ADR、開発ガイドに関する Markdown ドキュメント、およびテンプレートとする
- 本書の対象外は Java や YAML など実装ファイルの記述規約、ファイル形式の詳細設定、生成物そのものの実装ルールとする
- `docs/` のカテゴリは次のとおりとする

| 区分  | 役割                                      | 主な配置先      |
|-----|-----------------------------------------|------------|
| 入口  | `docs/` 全体の導線                           | `00_はじめに/` |
| 要件  | スコープ、関係者、業務ルール                          | `01_要件/`   |
| 設計  | ユースケース、ドメイン設計、API 設計、アーキテクチャ設計、データベース設計 | `02_設計/`   |
| 開発  | 開発フロー、規約、環境設定                           | `03_開発/`   |
| ADR | 採用判断の履歴                                 | `05_ADR/`  |

- 各階層の入口ファイル名は `README.md` とする
- 特に指定がない通常ドキュメントは日本語ファイル名を使用する
- 同じ種類のドキュメントは同じ語尾に揃える
- ドキュメント名の末尾は次のいずれかに寄せる
  - `概要`
  - `方針`
  - `ルール`
  - `ガイドライン`
  - `手順`
  - `テンプレート`
- ルールドキュメントは `xxx-rules.md` または `xxxルール.md` とする
- 概要ドキュメントは `xxx-overview.md` または `xxx概要.md` とする
- ガイドラインや手順書は `xxx-guideline.md` / `xxx-setup.md` または `xxxガイドライン.md` / `xxx手順.md` とする
- ADR は `ADR-0000-xxx.md` の形式とする

- 各階層には `README.md` を置き、役割と主要資料への入口を明記する
- 上位 README には概要と導線のみを書き、詳細説明は個別資料へ分ける
- 図や図の元データは `モデル/` または `ER図/` のような専用ディレクトリにまとめる
- テンプレートは `00_はじめに/テンプレート/` に置く
- ツール固有の補助資料は `補助資料/` `開発環境/` `IDE設定/` など用途別の下位ディレクトリへ寄せる
- 一時的な作業メモは正式ドキュメントと同じ階層に置かない
- 入口ドキュメント、正本、補助資料の役割が分かる配置にする

- 入口ドキュメントには概要、目的、導線を書く
- 詳細ドキュメントには具体的なルール、方針、手順を書く
- 実装変更で更新頻度が高い内容は、重複記述を避けて正本を 1 つに寄せる
- 補助資料は単独で正本に見えないよう、README や本文から関係を明記する
- カテゴリの README は簡潔に保ち、個別ドキュメントの置き場を示す
- Markdown は UTF-8 を前提とし、特別な理由がない限り ASCII を優先する
- 改行コードはプロジェクトの標準設定に従う

### 手順

1. `docs/README.md`、`00_はじめに/README.md`、本書の順に参照する
2. ドキュメントの目的と配置先を決める
3. ドキュメント種別に合わせてファイル名の語尾を決める
4. 特に指定がない場合は `00_はじめに/テンプレート/` 配下のテンプレートを使う
5. 必要なら `README.md` に導線を追加する
6. 関連する正本や補助資料との重複がないか確認する
