# 🌿 ブランチ命名規則

## 目的

ブランチ名を統一し、可読性と運用性を向上させる

---

## ルール

```
feature/<issue_number>
```

---

## issue_number

対象のissue番号

---

## 例

```
feature/12
feature/13
feature/14
```

---

## 命名ルール

- 英語
- 小文字

---

## 運用ルール

- 1issue = 1branch
- featureブランチはissue単位で作成

---

## 採用理由

- シンプルで分かりやすい
- Issue運用と相性が良い
- ブランチ名が短くなる
- 可読性が向上する
