# document-format-guideline

本資料は `docs/` 配下の markdown フォーマット統一ルールを定義します。

---

## 目的

- ドキュメントの読みやすさをそろえる
- 新規作成時の迷いを減らす
- README と詳細資料の書き分けを明確にする

---

## 基本ルール

- 文書タイトルは `#` を 1 つだけ使用する
- 大見出しは `##`、必要に応じて `###` を使う
- 見出しの粒度は浅く保つ
- 箇条書きは用途ごとにまとめる
- 長い説明は段落で書き、必要な箇所だけ箇条書きにする
- 区切りには `---` を使う
- ファイル冒頭に過剰な絵文字は付けない
- パス、コマンド、API はコード記法を使う

---

## README の書き方

各ディレクトリの `README.md` には以下を記載します。

1. そのディレクトリの目的
2. 管理資料の一覧
3. 更新ルール
4. 他ディレクトリとの境界

README には概要と入口のみを書き、詳細な設計内容は個別資料へ分離します。

---

## 個別資料の推奨テンプレート

```md
# タイトル

概要を1〜3行で記載する。

---

## 目的

- 何を定義する資料か
- 何の判断基準に使うか

---

## 内容

必要な項目を記載する。

---

## 補足

必要があれば関連資料や注意点を記載する。
```

---

## 命名ルール

- README は `README.md`
- 規約資料は `xxx-rules.md`
- 概要資料は `xxx-overview.md`
- 手順資料は `xxx-guideline.md` または `xxx-setup.md`
- ADR は `ADR-0000-xxx.md`

---

## 更新ルール

- 実装変更時は関連資料も同時に見直す
- 古い情報を残す場合は「現行ルール」と混在しないように明記する
- ルート README と `docs/README.md` は入口として常に最新に保つ
