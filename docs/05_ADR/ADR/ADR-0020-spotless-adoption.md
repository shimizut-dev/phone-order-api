# ADR-0020: Spotless を採用する

## ステータス

採用

## 日付

2026-05-06

---

## コンテキスト

phone-order-api では、フォーマット差分や import 整理漏れがレビューのノイズになりやすい。

Checkstyle は規約違反の検出には向いているが、整形そのものを自動適用する役割ではない。

このため、Java ソースの整形を自動化し、開発者ごとのフォーマット差異を抑える仕組みが必要である。

---

## 決定

Spotless を採用する。

`spotless-maven-plugin` を使用し、Java ソースに対して以下を適用する。

- `google-java-format`
- 未使用 import の除去
- annotation の整形
- 行末空白の除去
- 改行終端の統一

品質確認の基本コマンドは `./mvnw spotless:check test` とし、
CI でもこの流れを使用する。

---

## 理由

- フォーマット差分を機械的に揃えられる
- レビュー時の不要な指摘を減らせる
- `google-java-format` により整形基準を固定できる
- Checkstyle と役割分担しやすい

---

## 代替案

- IDE ごとのコードスタイル設定だけで整形を合わせる
- 手動整形を前提にレビューで差分を吸収する

---

## メリット

- 開発者ごとの差分を減らしやすい
- import 整理や空白調整の漏れを防げる
- CI とローカルで同じ整形基準を使える

---

## デメリット

- 整形ルールを細かくカスタマイズしにくい
- 初回導入時や大きな整形変更時に差分が増える
