# ADR-0017: Maven Enforcer Plugin を採用する

## ステータス

採用

## 日付

2026-05-06

---

## コンテキスト

phone-order-api では Maven をビルド基盤として採用している。

ローカル環境と CI で Java / Maven のバージョン差異があると、
手元では動作するが CI で失敗する、あるいは逆に CI 依存の状態になる問題が発生しやすい。

また、依存関係の重複定義は保守性を下げ、意図しない解決結果を招く可能性がある。

このため、ビルド前提を Maven 上で機械的に検証する必要がある。

---

## 決定

Maven Enforcer Plugin を採用する。

`maven-enforcer-plugin` を用いて、少なくとも以下をビルド時に検証する。

- Java の最低バージョン
- Maven の最低バージョン
- POM 上の重複依存バージョン定義

これらは `pom.xml` の `enforce-build-environment` で管理する。

---

## 理由

- 開発環境と CI の前提差異を早い段階で検出できる
- Java 21 / Maven 3.9.0 以上という前提を `pom.xml` に集約できる
- 依存関係の重複定義を機械的に防止できる
- ドキュメント記載だけに頼らず、ビルドで強制できる

---

## 代替案

- `README` や開発環境資料だけで必要バージョンを周知する
- CI の失敗を見てから個別に環境差異へ対応する

---

## メリット

- 環境差異による不具合を早期に発見できる
- チーム全体で同じビルド前提を維持しやすい
- Maven 利用時の品質ゲートを一元管理できる

---

## デメリット

- ルール変更時に `pom.xml` の更新が必要になる
- 新規参加者は環境要件を満たすまでビルドできない
